# 文本处理技能 (Text Processor)

## 技能描述
提供强大的文本处理功能，包括查找、替换、排序、去重、统计、编码转换等。

## 权限要求
- Level 2 及以上

## 可用命令

### 1. 文本查找
**命令**: `grep <模式> <文件>`

**说明**: 在文件中查找匹配指定模式的行。

**参数**:
- `模式`: 要查找的文本或正则表达式
- `文件`: 目标文件路径

**选项**:
- `-i`: 忽略大小写
- `-n`: 显示行号
- `-v`: 反向匹配（显示不匹配的行）

**示例**:
```
grep "error" log.txt
grep -n "TODO" source.c
grep -i "warning" app.log
```

---

### 2. 文本替换
**命令**: `sed <文件> <模式> <替换>`

**说明**: 在文件中替换指定的文本模式。

**参数**:
- `文件`: 目标文件
- `模式`: 要替换的文本模式
- `替换`: 替换后的文本

**示例**:
```
sed config.txt old_value new_value
sed document.htm .html .htm
```

---

### 3. 文本排序
**命令**: `sort <文件>`

**说明**: 对文件中的行进行排序。

**参数**:
- `文件`: 要排序的文件

**选项**:
- `-r`: 反向排序
- `-n`: 按数字排序
- `-u`: 排序后去重

**示例**:
```
sort names.txt
sort -rn numbers.txt
sort -u wordlist.txt
```

---

### 4. 去重
**命令**: `uniq <文件>`

**说明**: 移除文件中连续的重复行。

**参数**:
- `文件`: 要处理的文件

**选项**:
- `-c`: 显示重复次数
- `-d`: 只显示重复的行

**示例**:
```
uniq list.txt
uniq -c access.log
```

---

### 5. 文本统计
**命令**: `wc <文件>`

**说明**: 统计文件的行数、单词数和字符数。

**参数**:
- `文件`: 要统计的文件

**选项**:
- `-l`: 只统计行数
- `-w`: 只统计单词数
- `-c`: 只统计字符数

**示例**:
```
wc readme.md
wc -l source_code.c
wc -w essay.txt
```

---

### 6. 编码转换
**命令**: `iconv <文件> <源编码> <目标编码>`

**说明**: 转换文件的字符编码。

**参数**:
- `文件`: 要转换的文件
- `源编码`: 原始编码格式
- `目标编码`: 目标编码格式

**支持的编码**:
- UTF-8
- GBK / GB2312
- UTF-16
- ASCII

**示例**:
```
iconv chinese.txt GBK UTF-8
iconv data.csv UTF-8 UTF-16
```

---

### 7. 文本合并
**命令**: `merge <输出文件> <文件1> <文件2> ...`

**说明**: 将多个文件合并为一个文件。

**参数**:
- `输出文件`: 合并后的输出文件
- `文件1, 文件2...`: 要合并的文件列表

**示例**:
```
merge all.txt part1.txt part2.txt part3.txt
merge full_log.txt log1.txt log2.txt
```

---

### 8. 文件拆分
**命令**: `split <文件> <前缀> <行数>`

**说明**: 将大文件拆分为多个小文件。

**参数**:
- `文件`: 要拆分的大文件
- `前缀`: 输出文件前缀
- `行数`: 每个小文件的行数

**示例**:
```
split huge_log.txt chunk_ 1000
split big_data.txt part_ 5000
```

---

### 9. awk 文本处理
**命令**: `awk <文件> <表达式>`

**说明**: 使用 awk 表达式处理文本。

**参数**:
- `文件`: 目标文件
- `表达式`: awk 处理表达式

**示例**:
```
awk data.txt "{print $1}"
awk log.txt "{sum += $2} END {print sum}"
```

## 注意事项
1. 替换操作会直接修改原文件，建议先备份
2. 正则表达式语法遵循标准 POSIX 规范
3. 大文件处理可能需要较长时间
4. 编码转换前请确认源编码正确
